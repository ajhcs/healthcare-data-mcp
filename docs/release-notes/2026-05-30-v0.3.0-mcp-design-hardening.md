# v0.3.0 - MCP Design Hardening

Released: 2026-05-30

This release tightens the MCP surface around how agents actually discover,
select, call, recover from, and audit public healthcare-data tools. The project
already had broad source coverage and operator readiness; `v0.3.0` makes the
tooling more self-describing and safer to use from MCP clients.

The boundary remains the same: this project uses public and reference data. It
is not PHI infrastructure, does not establish HIPAA readiness, and does not
replace legal, compliance, clinical, or reimbursement review.

## Agent-Facing Tool UX

- Added required documentation sections for every checked-in `@mcp.tool`:
  discovery, when to use, parameters, returns, do/don't guidance, examples, and
  common mistakes.
- Added a pytest contract that parses server modules with AST and fails when a
  tool lacks the agent-facing documentation contract.
- Added capability clusters for broad servers including `public-records`,
  `workforce-analytics`, `financial-intelligence`, `discovery`, and
  `research-trials` so agents can choose smaller, task-aligned tool groups.

## Structured Recovery

- Extended shared MCP response helpers with error types, recoverability flags,
  structured data, fix hints, available options, and suggested tool calls.
- Added canonical helpers for invalid argument, not found, source unavailable,
  and policy denied responses.
- Added shared input normalization and mistake detection for CCN, NPI, state,
  ZCTA, FIPS, and catalog IDs.
- Migrated high-traffic recovery paths such as gateway dataset fetch,
  `cms-facility.get_facility`, and NPI search validation to return
  machine-parseable guidance instead of plain not-found messages.

## Resources and Observability

- Added standard per-server resources under
  `healthcare-data://server/{server_id}/...` for capabilities, datasets,
  examples, identity rules, and local metrics.
- Registered those standard resources on every server module.
- Added a lightweight non-secret observability decorator for MCP tools covering
  duration, outcome, error type, result size, dataset ID, and cache status.
- Exposed recent local tool metrics through the standard tooling metrics
  resource.

## Workflow Macros

- Added read-only macro tools in the discovery server for common operator
  workflows:
  `macro_quality_measure_lookup`,
  `macro_compliance_exclusion_screening`, and
  `macro_facility_profile_readiness`.
- Macros return bounded workflow plans, identity/evidence preservation rules,
  skipped or blocked source context, and next actions without hiding source
  caveats or executing multi-tool side effects.

## Web and Cache Hardening

- Hardened public web fetching by manually handling redirects, revalidating
  every redirect target, rejecting non-http(s) redirects, rejecting private or
  metadata hosts, bounding redirect depth, and blocking oversized HTML.
- Added atomic write helpers for bytes, text, JSON, DataFrame CSV, and Parquet
  outputs.
- Migrated runtime public-source cache writes across server and shared data
  loaders to atomic replacement where those artifacts are read as cache inputs.
