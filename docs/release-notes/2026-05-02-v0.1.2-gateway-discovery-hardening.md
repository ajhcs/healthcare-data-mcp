# v0.1.2 - Gateway and Discovery Hardening

Released: 2026-05-02

This release tightens MCP client discovery, corrects stale gateway metadata,
adds a separate authenticated live-data gateway, and cleans local tracker
artifacts out of the public source tree.

## MCP Gateway and Discovery

- Corrected remote `gateway` dataset metadata so advertised tool names match
  real registered MCP tools.
- Added `service-area`, `provider-enrollment`, `community-health`, and
  `research-trials` metadata documents to the remote-safe gateway.
- Added discovery tools for clients that prefer tool calls over MCP resources:
  `list_datasets`, `get_dataset`, `get_dataset_schema`,
  `get_dataset_source`, `get_cache_status`, `list_runbooks`, and `get_runbook`.

## Live Gateway

- Added `live-gateway` on port `8020` as a separate allowlisted router for live
  provider enrollment, hospital quality, claims analytics, LEIE/SAM exclusion,
  CDC PLACES, NIH RePORTER, and ClinicalTrials.gov tools.
- Local stdio remains usable without bearer auth.
- HTTP/SSE live-gateway transport requires credentials and rejects attempts to
  explicitly disable auth.

## Web Intelligence

- Added generic `search_web` and `fetch_web_page` tools to `web-intelligence`.
- `fetch_web_page` now rejects local, private, link-local, multicast,
  unspecified, and cloud metadata-service targets before fetching.

## Release Hygiene

- Bumped package and Desktop Extension metadata to `0.1.2`.
- Removed checked-in local issue-tracker exports and Beads workspace artifacts
  from the public tree.
- Updated README, client docs, Docker Compose, installer metadata, examples,
  Desktop Extension manifest, and packaging tests for `live-gateway`.
