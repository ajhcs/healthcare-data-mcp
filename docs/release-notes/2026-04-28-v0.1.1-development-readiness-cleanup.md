# v0.1.1 - Development Readiness Cleanup

Released: 2026-04-28

## Summary

This release prepares `healthcare-data-mcp` for the next public development cut by aligning package metadata, cleaning release artifacts, and tightening report-readiness workflows used by downstream healthcare market analysis.

## Release and Packaging

- Bumped the Python package and Claude Desktop Extension manifest to `0.1.1`.
- Consolidated release-prep work onto the development readiness branch.
- Cleaned public-release artifacts by removing extracted planning/prototype material from the packaged repository.
- Kept shipped MCP client examples generic, including the `SEC_USER_AGENT` example for `financial-intelligence`.

## Development Readiness

- Added Jefferson Health/LVHN reconciliation support for post-merger system profiles and report workflows.
- Added source-evidence validation for report-ingest fact rows so generated reports can reject rows without citation metadata.
- Added public cache acquisition support for provider enrollment data and improved manual cache status reporting.
- Added bounded municipal disclosure document handling and audited financial PDF parsing support for financial intelligence workflows.

## Price Transparency Consistency

- Fixed MRF registry CCNs for Jefferson/LVHN facilities so price-transparency search results match the canonical Jefferson facility ledger.
- Added coverage for the MRF CCN consistency path to guard against future registry drift.

## Known Limits

- This release does not add new live-source smoke-test requirements.
- External datasets still depend on upstream CMS, SEC, and related public-source availability.
