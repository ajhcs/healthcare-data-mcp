# v0.2.0 - Operator-Ready Productization

Released: 2026-05-22

This release turns `healthcare-data-mcp` from a broad public-data MCP toolkit into
a more opinionated, source-backed product that healthcare operators, analysts,
and agent users can install, validate, understand, and use for concrete public
data workflows.

The boundary remains deliberate: this project uses public and reference data.
It is not PHI infrastructure, does not establish HIPAA readiness, and does not
replace legal, compliance, clinical, or reimbursement review.

## Operator Readiness

- Added `hc-mcp doctor` as the single read-only health/readiness command.
- Doctor reports package/version, Python environment, dependency imports, server
  importability, local port conflicts, required and optional API-key posture,
  cache readiness, source freshness where known, client config hints, gateway
  posture, workflow readiness, registry drift, evidence-contract coverage, and
  distribution readiness.
- Added `hc-mcp doctor --check --json` for release gates, CI, and operator
  runbooks that need a machine-readable ready/not-ready result.

## Registry-Driven Product Surfaces

- Added a canonical typed server/capability registry with server ID, module,
  port, description, dataset IDs, environment keys, cache needs, zero-config
  eligibility, gateway exposure, preset membership, workflow roles, and safety
  notes.
- The registry now drives or validates the launcher, Docker Compose files,
  `.env.example`, installer/setup metadata, Codex/Claude/generic MCP client
  configs, MCPB/Desktop Extension choices, gateway/discovery metadata, README
  tables, source-ledger docs, and CI drift checks.
- Compose output binds local HTTP ports to `127.0.0.1` by default and documents
  the trusted reverse-proxy requirement for broader exposure.

## Evidence and Provenance Receipts

- Standardized healthcare-relevant tool results around shared evidence receipts
  with source name, source URL, dataset/source period, retrieval/cache metadata,
  cache status/freshness, match basis, confidence, caveats, and missing-data
  next steps.
- Expanded evidence-contract validation across the highest-value surfaces:
  hospital quality, provider enrollment, health-system profiling, financial
  intelligence, workforce analytics, public records, web intelligence, live
  gateway calls, and workflow planner outputs.
- Added shared nested evidence validation helpers so gateway and workflow
  surfaces can prove that returned facts retain provenance.

## Task-First Workflows

- Added executable workflow planning with `hc-mcp workflow <name>` and MCP
  discovery tools that return required sources, readiness, tool sequence,
  identity requirements, caveats, and report-ready fact-row templates.
- Added curated presets for compliance, market strategy, research, and
  metadata-only usage.
- Covered practical workflows including compliance/exclusion screening, facility
  profile, ownership/CHOW trace, hospital competitive profile, market/community
  health scan, quality measure lookup, research/trials activity profile,
  referral/leakage readiness, finance profile, quality profile, and system
  reconciliation.

## Healthcare Identity Map

- Added a shared identity-map foundation for joining CCN, NPI, PECOS enrollment
  IDs, AHRQ system IDs, owner IDs, aliases, ZIP/address fields, source-specific
  names, match decisions, confidence, conflicts, and unresolved identifiers.
- Integrated identity contracts into workflow plans and report-ingest contracts.
- Added cross-server MCP smoke coverage for the `system_reconciliation` workflow
  using Jefferson Health as a public known-entity example.

## Gateway and Remote Safety

- Kept `gateway` as a remote-safe metadata gateway and `live-gateway` as a
  separate authenticated policy gateway for approved live tools.
- Added live-gateway policy metadata for allowed tools, allowed scopes, request
  and result size bounds, rate-limit class, audit event shape, source caveat
  class, and provenance requirements.
- Added tests proving unsafe HTTP exposure and missing/invalid live-gateway auth
  are blocked, while local stdio development remains usable.

## Distribution and CI Gates

- Improved versioned install guidance for `pipx`, `uvx`, Python packages, Docker
  images, local stdio clients, local HTTP clients, and MCPB/Desktop Extension
  packaging.
- Added tagged container metadata checks and registry-rendered client artifacts
  for Codex, Claude Desktop, Claude Code/project MCP, and generic HTTP clients.
- Added CI/product readiness gates for secret scanning, dependency audit,
  installer dry runs, shell syntax, compile checks, registry-rendered artifacts,
  doctor readiness, workflow/preset smoke, MCP protocol smoke, MCP Inspector
  smoke, Docker zero-config startup, MCPB skeleton build, gateway auth tests,
  and Python distribution metadata checks.

## Public Data Expansion Included

This release also carries the public-data expansion that was previously tracked
under Unreleased:

- Public cache acquisition and status coverage for PHC4, AHRQ HFMD, PA hospital
  reports, NJ hospital public data, DE discharge data, and public cyber/breach
  sources.
- PHC4 public report indexing and extracted-table provenance for Hospital
  Performance, Financial Analysis, Common Procedures, and special public
  reports without using paid PHC4 datasets.
- Normalized PA/NJ/DE public state-health artifact models and query helpers for
  public financial health, staffing, productivity, throughput, and report
  provenance.
- Public financial health/community-benefit outputs from HCRIS, IRS 990
  Schedule H, and AHRQ HFMD while explicitly excluding HFMA MAP revenue-cycle
  KPI derivations and HCRIS days-in-A/R proxies.
- Cyber incident enrichment from OCR enforcement actions, SEC disclosures,
  searchable state breach notices, and CISA KEV vulnerability context with
  explicit incident confidence fields.
