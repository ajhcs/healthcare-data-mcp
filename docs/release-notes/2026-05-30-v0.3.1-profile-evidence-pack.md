# v0.3.1 - Profile Evidence Pack Workflow

Released: 2026-05-30

This patch release adds a read-only MCP workflow for Healthcare Toolkit
health-system profile population. The workflow returns structured public-data
evidence candidates and review findings; it does not write to Healthcare
Toolkit and does not estimate missing facts.

## Profile Evidence Packs

- Added health-system-profiler.build_profile_evidence_pack for state-bounded
  system profile evidence collection.
- Returns source-backed candidates for system identity, aliases, hospital
  rosters, CCNs, source-local identifiers, addresses, county/GEOID,
  coordinates, facility bed counts, system bed rollup guidance, affiliation
  evidence, official count claims, explicit conflicts, and unavailable-public
  findings.
- Every returned value carries evidence receipts, source metadata, dataset or
  source-family identity, source period, retrieval/access date, cache
  status/freshness, confidence, match basis, caveats, and MCP server/tool
  metadata.

## Source Handling

- Preserves explicit source precedence: CMS POS/HGI for facility identity,
  AHRQ as linkage context, HCRIS/state/reviewed official rows for bed
  corroboration, Census before OSM for geography, and PECOS/CHOW/reviewed
  official rows for affiliation review.
- Loads CMS cost-report/HCRIS rows through the existing hospital-quality cache
  path and PA state bed rows through state-health-data.
- Consumes official page/report evidence only from reviewed local cache rows;
  web-intelligence remains the collection path for new official/public pages.

## Workflow And Review Fixes

- Fixed profile_evidence_pack workflow planning so the cache-manager preflight
  step receives concrete workflow_id and inputs arguments.
- Moved PECOS/HCRIS/state/official enrichment after roster resolution so
  state + system_name calls use discovered CCNs for source lookups.
- Adds regression coverage for the stable JSON contract, cache-manager
  preflight, PA fixtures, bed/geography/affiliation resolution, exact official
  count claims, vague count review, and unavailable-public recovery hints.
