# MCP Tool Documentation Contract

Every @mcp.tool docstring must include the agent-facing sections below:

- Discovery
- When to use
- Parameters
- Returns
- Do / Don't
- Examples
- Common mistakes

The purpose is to put recovery and source-boundary guidance directly in the MCP tool surface where agents choose tools. Tool-specific prose should be preferred, but the shared baseline is acceptable for narrow tools when the function signature and structured response contracts carry the details.
