# Source Capability Ledger

This ledger states what the toolkit can support from public, source-backed
data today, what requires a local import or API key, and what must not be
asserted from adjacent records. It is a product contract for operators,
analysts, and agent workflows; it is not a claim of PHI, HIPAA, clinical, legal,
or eligibility-decision readiness.

Use this ledger with `hc-mcp doctor`, `hc-mcp workflow <name>`, and each tool's
`evidence`, `source_metadata`, caveat, match basis, and confidence fields.

## Operator Rules

- Prefer exact public identifiers first: CCN, NPI, PECOS enrollment ID, owner ID,
  AHRQ system ID, measure ID, UEI, or CAGE code when the source supports it.
- Treat names, addresses, ZIPs, sponsor names, owner names, and market labels as
  aliases or candidate match fields unless an exact identifier supports the join.
- Preserve every tool's `evidence` receipt and source caveat next to report-ready
  facts.
- Return source-status or import-required responses when data is missing. Do not
  convert absence into a negative factual claim unless the source contract
  supports that interpretation.
- Keep LEIE and SAM.gov outputs framed as screening support, not legal clearance
  or final identity verification.
